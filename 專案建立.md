## 專案目標

建立一個可維護、可測試、可透過 Docker 部署到 Zeabur 的 Node.js 後端 API。

技術要求：

- Node.js 22 以上
- TypeScript
- TypeScript strict mode
- Express
- Zod
- MySQL
- JWT Authentication
- Docker
- Docker Compose
- ESLint
- Prettier
- Vitest
- OpenAPI 3.1

新專案最終必須：

- 保留規格中的所有 `/api/v2/...` 路徑
- 保留原本 HTTP method
- 保留 Request 欄位名稱
- 保留 Params、Query、Body 結構
- 保留 Response JSON 欄位與資料型別
- 保留主要商業邏輯
- 保留現有資料庫 Table 與 Column 名稱
- 可以直接連接 Zeabur 現有 MySQL
- 不自動修改正式資料庫 Schema
- 可以透過 Docker 建立本機 MySQL 測試環境
- 使用新的 Node.js JWT 認證
- 不需要相容 Laravel Passport Token

## app.ts 與 server.ts

app.ts 只負責：

- Express app 建立
- security middleware
- CORS
- JSON parser
- request ID
- routes
- 404 handler
- global error handler

app.ts 不得執行 listen()。

server.ts 負責：

- 載入並驗證環境變數
- 初始化 logger
- 初始化資料庫
- 啟動 HTTP server
- 監聽 SIGTERM
- 監聽 SIGINT
- graceful shutdown
- 關閉 HTTP server
- 關閉 MySQL connection pool

## 資料庫方案

請先比較：

- mysql2
- Kysely
- Knex
- Prisma

依據以下條件選擇：

- 舊 MySQL Schema 相容性
- MyISAM 支援
- 不規則 Table 與 Column 命名
- Raw SQL 支援
- TypeScript 型別安全
- 不自動修改 Schema
- Zeabur 部署簡單度
- Node.js 22 相容性

優先採用簡單、可控、不會自動改 Schema 的方案。

請將選擇理由寫入根目錄 README.md。

不得直接把 migration-spec/node-migrations 當作正式 runtime 目錄。

可以參考其中邏輯，但正式檔案應放在：

- migrations/
- scripts/
- src/infrastructure/database/

## 資料庫安全限制

正式環境禁止：

- 自動 migrate
- 自動 sync schema
- ALTER
- DROP
- TRUNCATE
- 擅自加入 Foreign Key
- 擅自把 MyISAM 改成 InnoDB
- 擅自重新命名 Table 或 Column

本機 migration 使用：

migration-spec/sql/

正式環境只允許執行唯讀 schema verification。

## 認證

本階段只建立 Auth 模組與 middleware 骨架，不完成正式登入。

之後使用：

- Password Hash：優先評估 Argon2id，若要相容既有 bcrypt Hash 則使用 bcrypt
- Access Token：JWT
- Bearer Token
- Refresh Token：後續再決定

JWT 不得用來加密密碼。

## Zeabur 部署要求

應用程式必須：

- 使用 process.env.PORT
- 監聽 0.0.0.0
- 不硬編碼正式 Port
- 支援 reverse proxy
- 正確設定 trust proxy
- 正確處理 SIGTERM
- 提供 GET /health
- 提供 GET /ready
- 不依賴容器內永久檔案
- 不在 production start 時執行 migration
- 不使用 PM2，除非有明確必要理由

## Docker

Dockerfile 必須：

- 使用 Node.js 22 以上官方映像
- Multi-stage build
- 使用 lockfile reproducible install
- Production image 不包含 devDependencies
- 使用非 root user
- 不複製 .env
- 不在 build 或 start 時執行正式 migration
- 使用 exec form CMD

Docker Compose 只用於本機開發與測試，至少包含：

- api
- database

MySQL 使用：

- MySQL 8.0.33
- Named volume
- Healthcheck
- 環境變數提供帳密
- API container 不得使用 localhost 連資料庫
